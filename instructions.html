How to run the application?

  • 1. Ensure that you have Python>3.12 installed on your system.
  • 2. Clone the github repository LogSentinal
  • 3. Open a new terminal and navigate to the installed folder
  • 4. Run the following command on the terminal: python run.py
  • 5. After running the above command, the req dependencies would be installed and after that it would ask for the paths of the folders.
  • 6. Make sure the path you enter does not have quotes.

Open the dashboard at the given port

About the App:

  • The system can only detect and analyze Apache Log Files, Nginx Log Files and Linux Auth logs.
  • The application analyzes the files and gives the data about the following things:
    • Apache:
      • Suspicious IPs with req rate > 100 per min
      • Suspicious IPs with req rate > 1000 per min
      • Suspicious IPs based on No. of Downloads > 1000 per min
      • Suspicious IPs based on No. of Uploads > 1000 per min
      • Suspicious IPs based on No. of errors > 100 per min
      • SQL Injection Attempts
      • Suspicious IPs based on No. of 404 errors > 40% if the total requests
      • IPs sorted based on the bandwidth used.
    • Nginx:
      • Suspicious IPs with req rate > 100 per min
      • Suspicious IPs with req rate > 1000 per min
      • Suspicious IPs based on No. of Downloads > 1000 per min
      • Suspicious IPs based on No. of Uploads > 1000 per min
      • Suspicious IPs based on No. of errors > 100 per min
      • SQL Injection Attempts
      • Suspicious IPs based on No. of 404 errors > 40% if the total requests
      • IPs sorted based on the bandwidth used.
      • Requests with incomplete Responses.
    • Auth:
      • Suspicious IPs based on failed login attempts > 20 per min
      • No. of Failed Login attemtps by an IP.
  • The Dashboard will be running on the user's local host.
  • Real time monitoring starts only for the latest files and the new files that might be created during the monitoring.
  • If the folder has old files which have not been analyzed before then all they would be analyzed in the background during the monitoring.
  • Analyzed data is stored in the database for 30 days and can be accessed via the Historical tab of the dashboard
  • Summary report can be generated for the analyzed data. All the reports are stored in the SummaryReport folder.
